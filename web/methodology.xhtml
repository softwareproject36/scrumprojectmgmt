Methodology

1. PROJECT REGISTRATION AND PRODUCT BACKLOG PRESENTATION

Project Owner
Product owner
Scrum Master
Team (All)
Register the new projets
Registrer the product back log
Organize a planning meeting
Present the project and the backlog during the meeting
Set team members who will work on the project
Draw system context diagram and user case diagram
Plan for the first sprint planning meeting

2. SPRINT

Team (All)
Scrum Master
Analysts
Designers
Developpers
Testers
Discuss the items to be performed from the product backlog, define the sprint goal and functionalities
Divide the work into tasks
Assign tasks to the team members
Register sprint backlog
Draw PERT chart
Draw Gantt diagram
Follow burn-down chart
Supervise, monitor and report the progress
Prepare and hold daily scrum meeting
Design the system structure (class diagram, database schema etc)
Draw processes for selected user stories (activity diagrams and other diagrams if needed)
Help developers to write sql queries, classes and APIs
Design the system front end (User interface)
Help to develop user interfaces (pages, css files, Android activities, etc)
Create git repositories for all project components (web app, mobile app, sql queries, etc)
Clone repositories according to the assignments
Develop system functionalities
Push developed functionalities to the approprieted repository
Pull needed functionalities developed by others
Test new functionalities integrated in the system and the hole system
Report the results of tests
Help to design algorithms for functionalities and to develop libraries
Register finished tasks (each member)
Daily scrum meeting
Sprint review meeting
Sprint retrospective meeting
